Blog搭建日記

發表於2022-12-22|更新於2022-12-22|紀錄
|字數總計:1.7k|閱讀時長:5分鐘|閱讀量:

最近網課，突然看到一封網域自動續費的郵件，感覺這個網域只連結了MS的E5還有一個雲端硬碟挺浪費的。於是就決定搭個blog（？）

雖然有點莫名其妙但起因就是這個，總之，接下來就是過程

喔對了，由於我用的是macOS，所以本文為macOS下的操作過程，不過其他系統也可以參考（應該）

一、GitHub

1.註冊

應該沒人不會這個吧…

2.建立倉庫

在GitHub主頁點擊「New」新建倉庫

倉庫名稱為「用戶名.github.io」 描述隨便寫，也可以不填，建議點選「Add a README file」

（注意倉庫類型一定要選擇Public！！！）

其他的保持默認就行，將頁面拉到最下方點擊「Create repository」就將倉庫建立好啦！

目前Github上的操作暫時告一段落，接下來是安裝 Git

二、安裝Git

前往Git官網，點擊「Downloads」 選擇「macOS」 然後點擊 Binary installer 下的「installer」

然後點擊「Download」就行了，下載完成之後打開安裝包無腦下一步即可

打開 終端機 運行 git 得到下圖結果

Git 就安裝完成啦！

三、連結GitHub並克隆倉庫

1.連結GitHub

先在終端機內輸入 ssh 檢查是否安裝 SSH，得到下圖結果證明已安裝

接著輸入 ssh-keygen -t rsa 指定 RSA 算法生成密鑰，然後按四次Enter生成密鑰「id_rsa」和公鑰「id_rsa.pub」

然後按順序輸入以下內容

$ cd ~/.ssh
$ ls
$ cat id_rsa.pub

得到這樣的一大串內容，把被我打碼的這一串複拷貝下來

接著來到GitHub，點擊右上角頭像，接著點擊「Settings」 接著點擊「SSH and GPG keys」 然後點擊「New SSH key」

將剛剛拷貝的內容粘貼到「key」裏面，標題隨便填

最後點擊「Add SSH key」創建

回到終端機輸入 ssh -T [git@github.com](mailto:git@github.com) 檢驗是否成功連結

得到如下結果就說明連結成功啦

2.克隆倉庫

找個舒服的地方新建一個檔案夾 名稱隨意 最好是英文

在終端機中輸入 cd 然後將剛才創建的檔案夾拖到終端機窗口中，回車

按按照圖示順序點擊拷貝鏈接

接下來在終端機中輸入 git clone 剛剛拷貝的鏈接 將倉庫克隆到本地

確認本地檔案與GitHub上的檔案一致

接著下一步

四、安裝Node.js以及Hexo框架

1.安裝Node.js

前往 Node.js官網

下載完成無腦下一步安裝即可

打開終端機，運行 node -v 和 npm -v 檢查版本號驗證是否成功安裝

2.安裝Hexo框架

（過程中請保持終端機運行位置在剛剛創建的檔案夾）

打開終端機，輸入 cd 然後把剛剛 【三、2.】 中建立的檔案夾拖進終端機窗口，回車

輸入下面這條命令安裝 Hexo

安裝完成後，輸入 hexo init 初始化博客

然後輸入 hexo g 靜態部署

最後輸入 hexo s 即可在本地部署blog

瀏覽器輸入 http://localhost:4000 即可訪問在本地部署的網頁

在終端機內 control+c 終止運行

3.將Hexo部署到GitHub

回到剛剛安裝 Hexo 的檔案夾，打開「_config.yml」 拖到檔案最底部填上如下內容

deploy:
  type: git
  repository: https://github.com/exemple/exemple.github.io.git #倉庫地址
  branch: main

倉庫地址就是剛才【三、2.】時拷貝的鏈接

然後回到終端機中，運行下面的命令安裝 Git插件

安裝完成後按順序輸入以下命令

hexo clean
hexo g
hexo d        # `hexo g` 和 `hexo d` 可合併成 `hexo g -d`

成功以後，在瀏覽器輸入 https://exemple.github.io 就可以打開網頁了！

五、安裝主題

我在尋找茫茫多的主題之後，最終選擇主題的是最合我胃口的 Butterfly

還是一樣，在 Hexo根目錄 運行終端機，運行下面代碼安裝主題

git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly

然後修改 Hexo根目錄 中的「_config.yml」，將主題設置為 Butterfly

回到終端機，輸入以下代碼安裝pug以及stylus的渲染器

npm install hexo-renderer-pug hexo-renderer-stylus --save

最後，在 Hexo 的根目錄創建一個「_config.butterfly.yml」檔案，並把主題目錄的 「_config.yml」內容複製到「_config.butterfly.yml 」中（注意: 複製的是主題的 _config.yml」,而不是 Hexo 的 _config.yml）後面修改主題檔案的時候都是在「_config.butterfly.yml」檔案內操作

六、配置站點檔案

打開 Hexo根目錄 中的「_config.yml」，填寫站點內容

# Site
title: 葉！————————      #芝士雪豹（bushi）芝士標題，也就是Blog的名字
subtitle: 好位置          #芝士副標題，就是標籤頁上跟在標題後面的東西
description: 憨批          #芝士作者簡介
keywords: ''
author: 口十纟录亻本         #芝士作者名字
language: zh-TW            #芝士主題語言，顧名思義和主題的語言一樣
timezone: Asia/Shanghai    #芝士時區，我用的上海的

回到終端機，運行老三樣

hexo clean
hexo g -d
hexo s        #這條可以不用，只是部署稍微有點延遲，但這可以讓你在本地早點看到你這次整了些啥

Blog

成功啦！！

小結

我能小結什麼呢（）搭建過程非常坎坷，因為是新手所以幾乎把所有的坑都踩了一遍。我這裡沒把我踩過的坑放出來純粹是因為懶

零零總總花了六個多小時，達成了用終端機安裝 homebrew 然後一個小時沒安裝成功的痛苦，因為沒有cd導致hexo反覆報錯等等莫名其妙的成就，到最後，看到在瀏覽器輸入自己的網域，自己的站點就出現在顯示器上的時候，心裡五馬分屍，五穀雜糧，營養快線（bushi） 成語什麼不重要，反正你知道我要說什麼，總之我是很開心的

差不多這樣吧，第一次寫這麼長的東西，還蠻有成就感的

以後可能會在這個站點上面放很多東西，什麼類型的都會有，所以這個站點可能會變得很混沌（）導致我起名的時候花了超長時間的（（（
